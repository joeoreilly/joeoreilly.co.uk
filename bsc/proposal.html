Project Proposal

Location Aware Personal Assistant App

Project Aims

I aim to create an iOS App targeted against iOS 7 that will run on both the iPhone and iPad. The App will provide a user information based on their location, wether that be user defined data or automatically generated. This data could be, but isn’t limited to, any of the following; web links, documents, apps, address book contacts, attractions or travel information. The data will load only when the user’s device is within a predefined geographic area. Having spent my placement year writing iOS Apps, I am familiar with CocoaTouch. I haven’t, however, written any networking code and think that will be an interesting experience and could prove to be where a large slice of development time ends up being spent.

Tasks

  • Perform requirements elicitation
  • Create a detailed specification for the App to follow.
  • Create detailed UML class, use case, and state diagrams for the App.
  • Create a detailed development plan for the App.
  • Create user interfaces to add, manage and display user defined points of interest.
  • Create a user interface to view geographically relevant points of interest.
  • Create a user interface to view a point of interest in more detail.
  • Create an algorithm that parses various data sources for useful local information.

Deliverables

  • App specification
  • UML Diagrams
  • Development plan
  • Produce an App that shows predefined user data at a given location.
  • Produce an App that shows predefined user data for a given location alongside data automatically generated for the same location.
  • Submit the App to the iOS App store for Approval and sale.

Assessment Criteria

  • The specification should fully cover the App's feature set and data model.
  • The produced UML should be consistent with the App.
  • The App works on the iPhone
  • The App works on the iPad
  • The App has a functional user interface that allows managing of points of interest.
  • The App shows the correct predefined information at each/any location.
  • The App shows relevant automatically generated information at any location where data is available.
  • The App doesn't crash or exhibit strange behaviour, i.e. is relatively bug free

The Problem

Currently bookmarks, address book contacts, and other information available on smartphones and tablets is presented to a user in a static format that doesn't adapt to where the user is. This means users have to spend time searching for information that could be prepared and displayed for them based on their current location.

Why This Interests Me

I travel from home to University five days a week and find myself constantly switching between two 'contexts' which is currently hard to manage. Solving this with an App makes double sense, I'm using the device anyway and I worked as an iOS Developer in my placement.

How I choose this problem

I spent the last 12 months writing every idea, no matter how nutty, that came into my head down and this was the one I liked the most and had not seen anywhere else.
